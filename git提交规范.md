类型	适用场景
feat	新增功能（对应语义化版本的 MINOR 版本，如 1.1.0）
fix	修复 BUG（对应语义化版本的 PATCH 版本，如 1.0.1）
docs	仅修改文档（如 README、注释）
style	仅格式调整（如缩进、空格、命名规范，不影响代码逻辑）
refactor	代码重构（无新功能、无 BUG 修复，仅优化结构 / 性能）
perf	性能优化（如减少接口请求、优化循环逻辑）
test	新增 / 修改测试代码（如单元测试、集成测试）
chore	构建 / 工具链变更（如修改.gitignore、升级依赖、配置 CI/CD）
revert	回滚之前的提交（格式：revert: feat(auth): 新增记住密码功能）
